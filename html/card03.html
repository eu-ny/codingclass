카드 안에 이미지, 키워드란, 텍스트 그리고 사용자 프로필까지 넣어서 제작해보자!

< figma : 디자인 하기 >

이번 유형에서는 심화 버전인 만큼 그리드 할 요소가 많습니다.😂 차근차근 그리드를 만들어 카드형 페이지를 디자인 해봅시다!
이번 카드형 디자인은 카드 사이에 여백이 없기 때문에 라인을 넣어 카드 영역을 구분 해줍니다.

TIP ! 카드 섹션의 디자인이 동일할 경우 comtonent를 이용하면 효율적인 작업이 가능하다.
comtonent를 처음 적용시킨 카드가 원본가 되며 원본 comtonent를 수정하면 복제 comtonent들도 함께 수정되어 작업이 훨씬 수월해진다!

< HTML 소스 : 레아아웃 구조 파악하기 >

디자인한 웹 페이지 시안을 보며 페이지 구조를 파악한 후 웹 표준성과 접근성에 맞는시멘틱한 마크업을 해줍니다.

TIP ! 구조를 먼저 꼼꼼하게 파악하여 시멘틱한 마크업을 하는 것이 가장 중요하다. 때문에 복잡한 구조일 수록 구조를 잘 짜주어야 스타일이나 스크립트를 구현할 때 효율적으로 작업할 수 있다. 구조가 복잡할 수록 큰 덩어리부터 천천히 마크업하는 것이 좋다.(css 속성도 마찬가지!)
=> 순서는 컨테이너 부분까지 마크업한 다음 css 속성을 주어서 먼저 큰 레이아웃을 완성한다. 그 다음으로 큰 덩어리(+css)를 작업하는 것이 중요하다.

! html 주의사항 !
• img태그의 속성 중 alt안에 이미지에 대한 설명을 넣어줘야 시각 및 청각 장애인들도 스크린리더기를 통해 이해 가능
• img 태그와 같은 그림, 도표 자료 태그들은 figure 태그안에 넣어주는 것이 웹 표준을 준수한 것
• 웹표준과 웹 접근성, 호환성을 놓치지 않도록하자!

< CSS 소스 : 스타일 적용하기 >

구조 짜임의 순서대로 css 스타일을 적용시켜 봅시다. 이 때 디자인 시안을 참고하면서 html와 번갈아가며 큰 덩어리 부터 스타일을 적용시켜 나가는 것이 중요합니다.
속성에 대한 내용은 코드와 주석을 보면서 이해해봅시다!

TIP !
1. 먼저 전체 초기화(Reset)와 폰트 디폴트 설정을 해주는 것이 좋다.
2. 컨테이너 설정을 통해 메인 화면 크기와 여백을 설정해준 뒤 메인 콘텐츠인 섹션에게 스타일을 적용시킨다.
3. 카드 스타일은 동일하기 때문에 같은 클래스명을 넣어서 한 번에 속성을 설정해준다.
