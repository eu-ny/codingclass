#2. JAVASCRIPT : 스크립트로 페럴랙스 효과 주기

먼저 전체적인 흐름을 보고 스크립트가 어떻게 작동되는지 확인해봅시다! 🥹
오늘 스크립트에서 가장 중요한 포인트는 바로 재귀함수를 사용하여 스크립트를 짰다는 것입니다.

📖 재귀함수란?
재귀함수는 함수 안에서 자기 자신을 한 번 더 호출하는 함수를 의미합니다. 자신의 로직을 함수 안에서 반복하다가 일정 조건을 만족하면 함수 밖으로 나와 실행을 멈춥니다.
마지막 실행 조건을 설정하지 않은 경우 함수는 자기 자신을 무한대로 반복을 하게 되며 서버가 다운되기도 하니 꼭 조건을 설정해주어야 한다.

💡 전체 스크립트 보기 
//재귀함수 : 자기 자신을 한 번 더 실행. 즉 무한대로 실행시킨다. 이 때 그냥 실행시킬 경우 웹이 다운될 수 있다.
function scroll(){
    //반복문에 사용할 현재 스크롤 위치 값을 구하여 변수에 저장하기
    //세가지를 한 번에 작성한 이유는 호환성 문제로 인한 오류를 막기 위해서 이다. (앞의 값이 작동하지 않으면 뒤의 값으로 불러오기)
    let scrollTop = window.pageYOffset || document.documentElement.scrollTop || window.scrollY;

    //01
    //모든 섹션을 선택한 후 forEach문으로 반복시키기
    document.querySelectorAll(".content__item").forEach(item =< {
    
    //02    
    //현재 스크롤 값이 각 섹션의 offsetTop값 - window.innerHeight(사용자 화면에서 보이는 브라우저 영역) 값의 반 = 현재 보이는 브라우저의 반 높이의 이전 스크롤 값을 구할 수 있다.
    //즉 해당 섹션 스크롤 값에 도달하기 전에 그 이전 스크롤 값에서 에니메이션을 실행시키도록 하여 섹션 스크롤값에 도달 했을 때 완전한 섹션 콘텐츠를 확인할 수 있도록 한다.
    if(scrollTop > item.offsetTop - window.innerHeight/2){
        item.classList.add("show");
    }
    });
    //03
    //requestAnimationFrame() == 스크롤 이벤트 발생시키는 메서드 즉 스크립트에서 애니메이션을 구현할 때 자주 사용
    //requestAnimationFrame()은 브라우저에 최적화된 메서드이며 적절한 프레임 속도를 실행시킨다.(기본 1초 60프레임) 때문에 스크롤 이동 시 부드럽고 가벼운 움직임을 준다. 
    requestAnimationFrame(scroll);
}
scroll();
//이렇게 조건(메서드)을 설정한 후 실행시킬 경우 메모리를 적게 차지하여 웹이 다운되지 않는다.

1. 모든 섹션을 선택한 후 반복문 만들기
2. 반복문 안에 조건문을 통해 해당 섹션 영역의 반 정도의 이전 영역에 스크롤이 올 경우 에니메이션을 실행시켜 해당 섹션 영역에 스크롤이 올 경우 콘텐츠가 온전히 다 보이도록 설정합니다.
3. requestAnimationFrame()메서드를 통해 scroll 함수가 반복이 되고 마지막 애니메이션이 끝난 경우 반복을 중단하도록 합니다.
